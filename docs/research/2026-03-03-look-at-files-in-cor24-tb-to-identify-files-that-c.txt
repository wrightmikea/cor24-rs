  Key findings:

  Architecture Overview:
  - 24-bit RISC processor with 8 general-purpose registers (R0-R7)
  - 32 instructions with variable-length encoding (1, 2, or 4 bytes)
  - 24-bit address space with three regions: SRAM, embedded boot ROM, and I/O

  Source File Categories Identified:
  1. Architecture definition - cor24_cpu.v is the authoritative ISA reference
  (all 32 instructions implemented)
  2. I/O interfaces - cor24_io.v defines UART, LED/button, and interrupt
  controller
  3. 12 C demo programs - ranging from simple (hello, blinky) to complex (matrix
   multiplication, UART interrupts)
  4. Assembly code - uartaint.s shows calling conventions and interrupt handling
  5. Listing files - provide exact instruction encodings for validation
  6. Host tool - te.c terminal emulator for communicating with real hardware at
  921600 baud

  Hardware Interfaces Documented:
  - UART: 921600 baud, 8N1, hardware RTS/CTS flow control
  - GPIO: Single button input, single LED output at 0xFF0000
  - Interrupt controller: Single source (UART RX ready), automatic JAL r7,(r6)
  on interrupt

  Validation Strategy:
  - Unit tests per instruction category
  - Integration tests using the demo programs (fib.c returns 5702887, good
  recursion test)
  - The loadngo.c monitor program can load/execute code via UART - useful for
  dynamic testing

  Implementation Recommendations:
  - Start with interpreter approach, implement in phases
  - Phase 1: ALU + memory + branches → run hello.c
  - Phase 2: Full instruction set → run fib.c
  - Phase 3: UART I/O → run loadngo.c
  - Phase 4: Interrupts → run uartcint.c
