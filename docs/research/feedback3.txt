 most recent Claude analysis:

1) Line 4: it s NOT based on MachXO2, but MachXO (I may use MachXO2 in the future, but I am using MachXO now).

2) Line 15: splitting hairs perhaps, but there are not 32 instructions, there are 211 instructions. There are 32 operations, which, in combination with two other 3 bit fields, which usually denote registers, define the instruction.

3) Lines 19, 20: "add24ci" is used for the PC only, ordinary increments (+1) and relative branches (signed 8 bits). The "add24cico" module is the general purpose adder used for everything else. Funny that Claude didn't seem to read the comments. The reason these two adders need to be separate is that the instruction fetch stage is operating while the execute stage is doing whatever it does, they each need to do their work without grabbing exclusive access to a shared resource, which would lock out the other. This is not really necessary to understand the ISA however, that's more about how we execute the ISA, and the Fetch/Decode/Execute/Write-back pipeline.

4) Line 26: Fine point, I wouldn't expect anyone to know this unless they knew the implementation better, the block RAM is 3 KB long, but it is placed in the address space such that it could be 8 KB long, this is because I had considered using another part that had a 6 KB EBR, and I wanted most of this implementation to work the same with either part. Again, not critical to understanding.

5) Line 28: No, this is the test-bench for running the whole CPU in simulation. It isn't specific to any details of the SRAM access. There is a comment in this file, that Claude seems to have read, on the "sram" module used by this simulation, but the simulation clocks the whole SOC, the synchronous resets in cor24_soc, the CPU and I/O in cor24_cpu. Again, more detail than necessary to understand the ISA.

6) Line 60: "c" is condition, NOT carry

7) Line 89: The ISA is not described in this manual

8) Line 110: Only R0-R2 are general purpose. R3-R7 are special purpose.

9) Line 119: No. R5 is a register which is always zero, and can be accessed as "z" in compare instructions.

10) Line 129: There is 1 MB on-board the COR24-TB.

11) Line 130: FEE000-FEFFFF is an 8 KB range. On this FPGA, only the first 3 KB is there. Note the stack starts at FEEC00.

12) Line 147: Not sure what Claude means by auto-acknowledge, but the act of reading the data register does step the read index on the FIFO circular buffer, no separate I/O required to do that, if that's what it means, it is correct. Kind of amazing that the rest of this I/O description is correct, too.

13) Line 158: Claude still confused about the opcode/operation distinction, etc. Likewise in the instruction encoding description, below.

14) Line 293: this is NOT carry, but condition

15) Line 297: return address only in interrupt service
